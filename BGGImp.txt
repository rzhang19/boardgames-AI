Complete Implementation Plan for Feature #6 (BGG Integration)

Working BGG API Endpoints:
- Search: GET api.geekdo.com/api/geekitems?objecttype=thing&subtype=boardgame&search=<query>
  Returns: List of {name, objectid}
- Game Details: GET api.geekdo.com/api/geekitems?objecttype=thing&objectid=<id>
  Returns: Full data (name, description, players, playtime, images, categories, mechanics, BGG link)
- No authentication or API key needed -- just a User-Agent header
- No external dependency required (uses urllib + json from stdlib)

================================================================================

Step 1: Create club/bgg.py - BGG API Utility Module

- search_bgg(query) -- hits api.geekdo.com/api/geekitems?objecttype=thing&subtype=boardgame&search=<query>
  Returns list of {'id': int, 'name': str} (max 20)
- fetch_bgg_game(bgg_id) -- hits api.geekdo.com/api/geekitems?objecttype=thing&objectid=<id>
  Returns dict with: name, description, min_players, max_players, bgg_link, image_url, bgg_id
- Uses urllib + json from stdlib (no new dependencies)
- Graceful error handling (returns [] / None on failures)
- Uses short_description from BGG API instead of full HTML description

================================================================================

Step 2: Create club/tests/test_bgg.py - Tests for BGG API Module

Tests for search_bgg:
- Given a search query, when calling search_bgg, then a list of dicts with id and name is returned
- Given a search query with no matches, when calling search_bgg, then an empty list is returned
- Given a search returning many results, when calling search_bgg, then only first 20 are returned
- Given an API error, when calling search_bgg, then an empty list is returned
- Given a search query, when calling search_bgg, then the correct BGG API URL is called

Tests for fetch_bgg_game:
- Given a valid BGG ID, when calling fetch_bgg_game, then game data dict is returned
- Given a BGG game with minimal data, when calling fetch_bgg_game, then None is used for missing fields
- Given an API error, when calling fetch_bgg_game, then None is returned
- Given a game with HTML description, when calling fetch_bgg_game, then short_description is used as fallback

All tests use mocked urllib calls (no real network requests)

Step 3: Update BoardGame Model in club/models.py

Add the following fields to the BoardGame model:
- bgg_id: PositiveIntegerField(null=True, blank=True) -- stores the BGG object ID
- bgg_link: URLField(blank=True) -- stores canonical BGG URL
- image_url: URLField(blank=True) -- stores BGG thumbnail image URL
- bgg_last_synced: DateTimeField(null=True, blank=True) -- when data was last pulled from BGG
