- understand expected behavior based on criteria given during prompting

- if any tests fail, stop the generation so I can review

### DON'T
- do not add any heavy dependencies without approval
- do not modify (add/edit/remove) any files outside of this directory

### Permissions
Allowed without prompting:
- read files
- list files

Ask first before acting:
- package installs
- git push
- deleting files, chmod
- running full build or end to end suites

### When editing text documents (design documents)
- ALWAYS ask for permission before editing
- Display a preview of the changes before making any changes, and prompt for permission

### When working on new features (ExtraFeatures.txt or Jiras)
- Always work on one new feature at a time. If explicitly stated, work on each number one at a time. Ask if one number has multiple subpoints, if to work on them separately or together.
- When complete, at the end of tests completing, mark the line as DONE, as stated in the file instructions (usually near the top of the file)

### Regarding tests:
- Upon working on a feature, always check if there are existing tests that can be reused instead of creating duplicate tests.
- For end to end system tests, always check if we can incorporate tests into an existing system test instead of creating a net new test that would use previous setup or transactions.
